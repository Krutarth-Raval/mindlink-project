1. BIG PICTURE — What happens when a user clicks "Generate"?
================================================================================

Here is the full journey from button-click to result displayed on screen:

  [User types a prompt]
        |
        v
  [Clicks "Generate" button]
        |
        v
  [Frontend checks: Do I have enough credits?]
   - Image costs 1 credit
   - Video costs 5 credits
   - If not enough credits → Redirect to /buy-credits?redirect=/ai-generation
        |
        v
  [Frontend calls the Server Action: generateAIContent({ prompt, type })]
        |
        v
  [Server Action (generation.js) runs on the SERVER]
        |
        v
  [Server checks Clerk authentication & Database credit balance]
        |
        v
  --- IF type === "image" ----------------------------------------
  |   [Call HuggingFace API → get image Blob → convert to base64]
  |   [Deduct 1 credit from database]
  ----------------------------------------------------------------
        |
  --- IF type === "video" ----------------------------------------
  |   Step 1: POST to RunwayML API (Gen-4.5) → get back a task_id
  |   Step 2: Poll RunwayML every 5-10 seconds until status = "succeeded"
  |   Step 3: Get the video URL from the results array
  |   [Deduct 5 credits from database]
  ----------------------------------------------------------------
        |
        v
  [Frontend receives result → UI re-renders → Credits refreshed via router.refresh()]

2. THE FRONTEND (Dynamic Redirects & Studio)
================================================================================

The platform uses `generation-studio.jsx` as the primary workspace.

Smart Redirection:
  - If a user tries to generate a video but has 0 credits, they aren't just 
    showing an error. 
  - The system redirects them to `/buy-credits?redirect=/ai-generation`.
  - After a successful purchase via Razorpay, the buy-credits page uses the 
    `redirect` param to automatically send the user back to the studio.

Performance:
  - Uses `next/script` for Razorpay to ensure fast initial page loads.
  - Video rendering uses a real-time progress state to keep the user engaged.

================================================================================
3. THE BACKEND SERVER ACTION (RunwayML & HuggingFace)
================================================================================

Located in `actions/generation.js`. 

Security:
  - Marked with "use server" to hide API keys (RUNWAY_API_KEY, HF_TOKEN) 
    from the browser.
  - Validates session with Clerk's `auth()` on every request.

Atomic Credits:
  - Uses Prisma's native `{ increment/decrement }` to prevent "double spending"
    if a user clicks generate twice very fast.

4. IMAGE GENERATION — Step by Step
================================================================================

  Provider : HuggingFace Inference API
  Model    : Tongyi-MAI/Z-Image-Turbo
  Cost     : 1 credit
  Process  : 
    1. Send prompt to HuggingFace.
    2. Receive raw binary data (Blob).
    3. Convert to Base64 String so the <img> tag can show it instantly.
    4. Deduct 1 credit.

================================================================================
5. VIDEO GENERATION — Step by Step (RunwayML Gen-4.5)
================================================================================

  Provider : RunwayML
  Model    : gen4.5 (State-of-the-Art Cinematic)
  Cost     : 5 credits
  Process  :
    1. Submission: POST to `https://api.dev.runwayml.com/v1/tasks` with model specs.
    2. Polling: The server enters a loop, querying the task ID.
    3. Completion: When status is "succeeded", the server extracts the video URL.
    4. Deduction: 5 credits are removed from the User's account.

================================================================================
6. CREDIT SYSTEM — Razorpay & Automations
================================================================================

MindLink uses a professional payment gateway integration:

  Flow:
    1. User selects a plan (Starter, Premium, Enterprise).
    2. Frontend calls `/api/payments/create-order` to get a Razorpay Order ID.
    3. Razorpay Checkout modal opens.
    4. Upon success, `/api/payments/verify` validates the cryptographic signature.
    5. Credits are added, and a record is created in `PaymentHistory`.

  Webhooks:
    - Located at `/api/payments/webhook`.
    - Automatically captures payments even if the user closes their browser 
      too early.
    - Uses `payment.captured` event to ensure the user gets what they paid for.

7. ERROR HANDLING & REDIRECTS
================================================================================

  - API Errors: Normalized to show friendly text (e.g., "AI quota exceeded" 
    instead of raw JSON).
  - Build Errors: Fixed by moving Razorpay initialization inside route handlers
    so they don't break during Vercel deployment.
  - Insufficient Credits: Automatically redirects to the pricing page.
